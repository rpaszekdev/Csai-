1. Three Schools of Machine Learning

The lecture frames itself with Yann LeCun's cake analogy: machine learning is a cake — reinforcement learning is the cherry on top, supervised learning is the icing, and unsupervised learning is the actual cake. The bulk of structure that has to be discovered in the world is unlabeled.Prof

Most ML courses (Stanford, Tilburg, etc.) heavily favour supervised learning. Kirtay points out there is rarely a course dedicated entirely to unsupervised learning, even though for roboticists it is the most relevant family. This whole lecture is the corrective.

Definition · core Exam

Unsupervised learning differs from supervised and reinforcement learning:

  • The aim is not to map input → output by minimizing a cost function.
  • There is no explicit reward function to tune the behaviour of the network.
  • There is no explicit score / metric to judge "how good" a behaviour is.

Three-way comparison Exam

SchoolSignalGoalExample
Supervised Labeled (x, y) pairs Learn a function f(x)→y by minimizing a loss CNN image classification (ImageNet, AlexNet)
Reinforcement Scalar reward from the environment Learn a policy π(s)→a that maximizes return Robot trying actions, getting +10 / −10
Unsupervised No labels, no reward Discover structure (clusters, prototypes, manifolds) K-means, competitive learning, SOM

The motivation is biological too. Humans learn from observation: when somebody touches a hot stove, you don't need a label "this is a hot stove" — you infer the structure. Babies are not given labelled environments. Neither are robots in the wild.

2. Why Unsupervised Matters for Robots

For robots, three pressures push us toward unsupervised methods:

When to use unsupervised Exam
  1. We have a lot of sensory data and we cannot hand-label it.
  2. We do not have a reward function to guide behaviour.
  3. We cannot construct a score / loss function to train the network.

The data deluge

A robot has a camera, distance sensors, sonar, photoresistors, position encoders, IMUs, force/torque sensors, and increasingly an artificial skin (e.g. iCub has tactile patches with thousands of taxels). If a robot just hangs around for 20 minutes, the volume of recorded frames and sensor readings is enormous — labelling each frame ("there is a person in this image, there is a chair in this image") is impossible.Prof

No reward in the wild

RL needs an operator-assigned reward (hit obstacle = −10, reach goal = +10). Outside curated tasks, there is no such signal. Unsupervised methods extract structure without a reward function.

Cost of error

RL is also physically expensive — letting a real robot try random actions can damage the robot or hurt people. So we collect raw sensory data, feed it through unsupervised methods, and discover useful prototypes safely.

Prof example Prof

"You want a robot to grasp a phone — should it use the left hand or right hand?" Initially the robot doesn't know. You collect tactile / position samples while it manipulates objects, then cluster the data. The red cluster ends up as left-handed objects, the blue cluster as right-handed. No labels were ever provided.

3. K-Means Clustering

You met K-means already in HCI for persona segmentation. Here it returns as the entry-level unsupervised algorithm.

The name decomposes: K = number of clusters, means = the cluster centres are updated to the mean of their assigned points.

K-means · pseudocode Exam
  1. Choose number of clusters K.
  2. Set cluster centres randomly.
  3. Until convergence:
    • For each input: compute distance to every cluster centre; assign the input to the closest centre.
    • For each cluster centre: move the centre to the mean of the inputs assigned to that cluster.

Convergence criteria

  • Fixed number of iterations — simplest. Stop after N steps.
  • Threshold on movement — stop once centres move less than ε (e.g. 0.05 in x and y) between iterations.

Kirtay walked the class through a live visualization: with K=2 the centres start randomly, then "snap" toward the dense regions until they sit roughly in the middle of each cluster. He called this out as exam material.Exam

Choosing K — how do you pick the number of clusters?

  • Visualize the data first.
  • Domain knowledge — e.g. you know the robot has two hands, so K=2.
  • Run for several K (2, 3, 4, …) and pick the best.
  • Elbow method and silhouette score are formal choices (mentioned, not derived).

Mean vs median — the noise problem Exam

K-means uses the mean, which is sensitive to outliers. Example given in lecture: salaries 30, 20, 10, 40, then add a single 1000. The mean shifts to 220 — a number that doesn't even exist in the dataset. If the robot's downstream decision depends on this average, the cluster centre is now wrong.

Mean vs median complexity Exam

Mean is robust to compute but sensitive to outliers: complexity O(n) — sum + divide.

Median is robust to outliers (sort then pick middle) but more expensive: complexity O(n log n) on unsorted data.

So K-means stays with the mean because n log n > n; you trade noise robustness for speed. Use median only if you know your data has heavy outliers.

Practical advice from the slide: eliminate outliers before running K-means, or move to k-medoids / k-medians when the dataset is dirty.

4. Competitive Learning

Competitive learning is the bridge from K-means to the brain-inspired models. Same paradigm: no cost function, no reward function. The twist is that the "cluster centres" are now the weight vectors of neurons, and the neurons compete to represent the input.

Competitive learning · core idea Exam

Form a unit (or neuron) which becomes activated for similar inputs. The activation of a neuron increases as the distance between its weight vector and the input vector decreases.

Supervised vs competitive — a single picture

In supervised learning, an input vector [0.42, 0.12, …, 0.18] is run through forward + backward passes, with a label and a loss. In competitive learning, the same input vector is just compared (by Euclidean distance) against each unit's weight vector — no label, no loss, no backprop.

Winner-take-all algorithm

Pseudocode · WTA Exam
  1. Initialize the weights for each unit.
  2. For each input, compute the Euclidean distance from the input vector to each unit.
  3. Select the unit with the smallest Euclidean distance — this is the winner unit.
  4. Update only the weights of the winner unit: w ← w + α (x − w).

α (alpha) is the learning rate, α > 0. The size of the input vector and the unit weight vector must be the same.

The role of the learning rate Exam

  • α = 0 → weights never update. The network is frozen and learns nothing.
  • α = 1 → the winner's weight vector jumps directly onto the latest input. Catastrophic forgetting — every new sample overwrites whatever was learned before.
  • 0 < α < 1 → the weight vector slides smoothly toward the input. In SOM, α is also annealed (reduced over iterations) so the map stabilizes.

What competitive learning gives you

The weight vectors become prototypes. The training process moves these prototypes toward the input distribution. Useful for:

  • Clustering the input.
  • Vector quantization — compress a huge input space into a small codebook of prototypes.
  • Feature extraction — extract features in an unsupervised way.
5. Self-Organizing Maps (SOM / Kohonen)

The Self-Organizing Map — also called the Kohonen map — is a variation of competitive learning. It is the headline algorithm of this lecture and the most exam-worthy structure to memorize.Exam

SOM · structural definition Exam
  • Self-organizing map is a variation of competitive learning.
  • Units are arranged on a 1-D string-like or 2-D lattice-like structure (a grid of neurons).
  • Units that are close to each other in the lattice respond to similar inputs and end up with similar weights.

The lattice itself is the magic. K-means and basic competitive learning have independent cluster centres — moving one says nothing about the others. In a SOM, neurons have a topological neighbourhood: when the winner updates, its neighbours update too (more weakly). The result is a topology-preserving map — similar inputs activate nearby neurons on the grid.

SOM algorithm — winner-take-all plus neighbourhood

SOM · pseudocode Exam
  1. Initialize the weights for each unit.
  2. For each input, compute the Euclidean distance for every unit.
  3. Select the unit with the smallest distance — the winner / Best Matching Unit (BMU).
  4. Update the winner's weights: w_p ← w_p + α (x − w_p).
  5. Update the neighbours' weights too, scaled by a neighbourhood function damp(i, p): w_i ← w_i + α · damp(i, p) · (x − w_i).

Notes: α is a learning rate > 0, reduced over iterations. The neighbourhood radius is also typically reduced over time. Input size and unit weight size must match.

What a trained SOM looks like

After training, the lattice has self-organized: similar input vectors fire neurons in the same region of the grid. Classic example — feed a SOM a bunch of documents; the trained map clusters similar document contents into nearby neurons (a 2D semantic atlas you can browse with your eyes).Prof

Limitations and variations Exam

  • Sensitive to weight initialization — different random seeds can produce significantly different final maps.
  • Training duration grows with the dimensions of the network (bigger lattice = slower training).
  • Different SOMs can find different similarities for the same inputs — the map is not unique.
6. K-Means vs Competitive vs SOM

All three are unsupervised "find-the-groups" algorithms. They differ in what they store and how they update.

PropertyK-MeansCompetitive LearningSOM
What is learned K cluster centres (means) Weight vectors of N units Weight vectors on a lattice
Update rule Centre ← mean of assigned points Winner shifts toward input by α Winner + neighbours shift toward input
Topology preserved? No No Yes — neighbour units stay similar
Output dimensionality K labels N prototypes 2-D map you can visualize
Outlier sensitivity High (mean) Lower (single-step shift) Lower (annealed α + neighbourhood smoothing)
Robotic use case Quick segmentation, prototyping Online vector quantization Multimodal sensor mapping, ERA, SOIMA
Mental model

K-means → "where are the centres?" Competitive → "which neuron wins?" SOM → "which neuron wins, and how does the map deform around it?" Each step adds structure.

7. SOM-Based Architectures for Robots

Why does Kirtay spend so much time on SOMs? Because SOMs are the building block of several developmental robot architectures. Two are name-checked explicitly.

Epigenetic Robot Architecture (ERA) Exam

ERA — epigenetic == developmental
  • The units of the architecture are SOMs.
  • Each SOM processes a different modality (vision, touch, proprioception, audio…).
  • Because the modalities are wired into a shared system, a missing modality can be reconstructed from the others.
  • An open-source Python library (PyERA) exists.
  • Source: Morse, Greeff, Belpaeme & Cangelosi (2010).

SOIMA — Self-Organized Internal Models Architecture Exam

SOIMA — sensory-motor schemes
  • Units are again SOMs.
  • Implements both forward and inverse models in the same architecture — i.e. predict next sensor state from action, and predict action needed to reach a sensor state.
  • Supports missing-modality reconstruction.
  • Performs sensorimotor mapping — the bridge from raw sensor flow to motor commands without labels.
  • Source: Hermosillo-Valadez (2016), Frontiers in Robotics and AI.

Beyond these, the slides reference a paper on distributed Growing Self-Organizing Maps for unsupervised skill transfer between autonomous robots — i.e. one robot teaches another a skill without labelled training data. Worth a glance for projects.

Why this matters for the exam Exam

If the exam asks "name a SOM-based robotic architecture and what it does", you should be able to answer ERA (multimodal, missing modality reconstruction) or SOIMA (forward + inverse models in one place). These are concrete examples of unsupervised learning solving a real robotics problem.

Connect the dots with Lecture 4 (Embodiment): SOM-based architectures are how a robot self-organizes its sensorimotor experience without a labelled curriculum — the algorithmic side of "form follows function".

8. Tooling & Practical Tips

The frameworks for unsupervised learning are not as mature as those for deep learning or RL — Kirtay is honest about this. Here is the working stack he recommends:

LibraryUse for
scikit-learn K-means, mini-batch K-means, mean-shift, plus dimensionality reduction (PCA, t-SNE). Default toolbox.
TensorFlow Customized unsupervised methods — autoencoders, custom competitive layers, anything you have to roll yourself.
MiniSom Generic self-organizing map implementation — drop-in for SOM experiments.
PyERA Implementation of the Epigenetic Robotic Architecture with object-oriented SOM units. Use this if you want stacked / multimodal SOMs.

scikit-learn's algorithm decision tree Exam

Kirtay walked through the official scikit-learn cheat sheet:

  1. < 50 samples? Don't run any learning algorithm — fit it in memory, use a lookup table with hard-coded actions.
  2. More data? Are you predicting a category (door open / closed) or a quantity (joint angle for a stop gesture)?
  3. Categorical + labelled? Use a supervised classifier.
  4. Categorical + unlabelled? Move to unsupervised.
  5. Number of clusters known? Use K-means.
  6. Number of clusters unknown? Use mean-shift or mini-batch K-means.

Other unsupervised methods (mentioned but not deep-dived)

  • Hopfield networks — auto-associative memories. Train with audio + visual; at test time, present visual only and recover the associated audio.
  • Hierarchical Temporal Memory (HTM) — neuroscience-inspired model for extracting temporal correlations in input streams.
  • Autoencoders — multi-layer perceptron with a bottleneck; trained to reproduce the input. Once trained, the bottleneck is a compressed latent space useful as features.
  • Visualization methods — PCA, t-SNE, UMAP — useful for looking at high-dimensional sensor data, not the focus of this lecture.

Key Takeaways
  1. Three schools of ML: supervised (labels + loss), reinforcement (reward), unsupervised (neither). LeCun's cake: unsupervised is the cake.
  2. Unsupervised motivation for robots: too much sensory data to label, no reward function in the wild, no constructible loss. Three bullets — likely an exam item.
  3. K-means pseudocode: pick K, place centres randomly, repeat {assign each input to nearest centre, move centre to mean of its inputs} until convergence. Convergence = fixed iterations or centre movement < threshold.
  4. Mean vs median: mean is O(n) but sensitive to outliers; median is O(n log n) but robust. K-means uses mean; eliminate outliers first.
  5. Competitive learning: winner-take-all. Compute Euclidean distance from input to every unit's weight vector; only the winner updates: w ← w + α(x − w). α=0 freezes, α=1 forgets everything.
  6. SOM = competitive learning + topology. Units sit on a 1-D string or 2-D lattice. Update winner AND neighbours, scaled by a neighbourhood function. Result: similar inputs land on nearby grid units (topology-preserving map).
  7. SOM limitations: sensitive to initialization, training scales with lattice size, different SOMs can find different similarities — the map is not unique.
  8. SOM-based robot architectures: ERA (epigenetic = developmental, multimodal, missing-modality reconstruction) and SOIMA (forward + inverse models, sensorimotor mapping). Concrete answers if asked "where is unsupervised learning used in robotics?".
  9. Tooling: scikit-learn for K-means + dim. reduction, MiniSom for SOMs, PyERA for stacked architectures, TensorFlow for custom autoencoders. The space is less mature than DL / RL.
  10. Mental hierarchy: K-means → competitive learning → SOM. Each step adds structure (means → weight vectors → topology). Memorize the pseudocodes — they are the most likely exam targets.
