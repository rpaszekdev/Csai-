2. What Is an Autonomous System?
Definition · core Exam

Autonomous [robot] systems are systems capable of operating:

  1. without human intervention
  2. for prolonged periods of time
  3. in partially unknown environmental conditions

(Source: Nolfi — Behavioral and Cognitive Robotics: An Adaptive Perspective). The course follows this definition. All three conditions must be satisfied.

Why this matters: many systems pass one condition (a thermostat operates without intervention) but fail the others. The combination is what separates a control system from an autonomous one.

Final-exam framing question

Final exam · sample Exam

Prof. James Spigler claims: "a simulated agent that works in a fully known environment, e.g., a factory environment, with a predefined set of actions, is an example of an autonomous system."

Agree or disagree, and explain.

Disagree. Spigler's setup violates condition (3) — a fully known environment is not partially unknown. A predefined action set with no adaptation also strains condition (1) operationally — the human pre-specified every behavior. The agent is automated, not autonomous.

3. What Makes a Robot a Robot?

An artificial system (robot) is defined by six elements:

  1. Body — physical structure
  2. Sensors — perceive the environment
  3. Actuators — act on the environment
  4. Brain (controller) — maps sensor input → actuator output
  5. Environment — physical and possibly social
  6. Function — exhibits behavior performing a task
Prof emphasis Prof

The course considers simulated robots as virtually embodied robots — YES, they are robots. But unless stated otherwise, "robot" defaults to a physically embodied robot.

4. The Body
  • Physical structure of the robot.
  • Task & environment drive design — "form follows function" (HCI principle).
  • Material spectrum: rigid (most industrial robots) ↔ soft (compliant, safer for human contact).
5. Sensors

A sensor is "a device that measures some physical quantity in the environment".

Examples: cameras, lidar / laser / IR rangefinders, microphones, touch sensors, position encoders.

Two categories Exam

TypeMeasuresExamples
Proprioceptive Internal state of the robot itself Joint position, joint angle, battery level
Exteroceptive Quantities external to the robot Camera, tactile sensor, microphone

Sensors can also be simulated in virtual environments — you'll do this in the practicals.

6. Actuators

Actuators convert stored energy into movement, allowing the robot to move and modify the environment.

  • Electric motors — most common
  • Pneumatic actuators — compressed air; high force, low precision
  • Soft actuators & active material objects — emerging field
7. The Brain (Controller)
  • The brain regulates the state of the actuators using current and previous sensor states.
  • Typically a computer embedded in the robot's body.
  • Can also live outside the robot — cloud-enabled robotics.
  • Function the controller enables, e.g.: wall-following.
Prof challenge Prof

Do we really need a brain to generate behavior?

The provocation comes from passive walkers — robots that walk down a slight incline using only gravity and clever leg geometry, no controller at all. Embodiment can substitute for computation. This thread continues in Week 04 (Embodiment).

Key Takeaways
  1. Definition (3 conditions): autonomous = no human intervention + prolonged operation + partially unknown environment. Miss any one → not autonomous.
  2. Robot anatomy: body, sensors, actuators, brain, environment, function — six pieces.
  3. Sensor split: proprioceptive (internal state) vs exteroceptive (external state) — easy fill-in-the-blank target.
  4. Brain ≠ always required. Passive walkers show embodiment can replace computation; foreshadows Week 04.
  5. Simulated robots count in this course, but unless stated, "robot" = physically embodied.
  6. Pass requirements: exam ≥ 5.5 AND project deliverables submitted AND demo day attended.
  7. LLM use is allowed if disclosed and you can explain every part. Silent use = Examination Board referral.
  8. Spigler-style exam question tests whether you can apply the 3-condition definition to a counter-example.
