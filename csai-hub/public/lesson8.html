Lesson Eight · the finale

Teaching machines to see — still unsolved.

In 1966, MIT thought a summer project could "solve vision". Sixty years later, we have CNNs, transformers, state-space models, and we still can't match a toddler. This lesson is a tour of where the field is now and where it's heading.

Slides by Silvia Pintea
Department of Cognitive Sciences and AI · Tilburg University
Section 1 · slides 1–3 · Papert 1966

The summer that wasn't enough

In July 1966, Seymour Papert at MIT wrote a now-famous internal memo: The Summer Vision Project. The goal was to assign undergraduates a few weeks each, divide the problem into "figure-ground" and "object recognition" pieces, and have a working visual system by September. The proposal still lives in MIT's archive (DSpace 1721.1/6125). It was wrong by about sixty years and counting.

Why is it so hard? Because pixels are not meaning. A digital image is just a grid of brightness values. Between that grid and the answer "this is a cat" sits an enormous gap: lighting changes the numbers, viewpoint changes them again, occlusion hides parts, scale blows them up or down, deformation reshapes them, and intra-class variation means two different cats share almost no pixels.

The pixels on the right are pulsing through a "feature extractor". Each pulse is a filter response — the network slowly building structure out of brightness alone.

The semantic gap. A 224×224 RGB image has about 150,000 numbers. The label "cat" is one symbol. The job of any vision system is to throw away almost all of those numbers without throwing away the cat.

Quick check

Why did Papert's summer project fail to solve vision?

Speed was a real bottleneck, but the deeper issue is the semantic gap. Even with infinite compute, a single straight line of pixel logic cannot turn brightness into "cat". You need representations that survive lighting, viewpoint, occlusion and scale.

Section 2 · slides 13–15

The task zoo

"Computer vision" is not one task. It is a family. Each task takes the same input — an image — and asks for a different shape of output. The shape of the output decides the shape of the network and, more importantly, the shape of the loss.

The classic line-up:

  • Classification — one label per image. Is this a cat?
  • Object detection — labels plus bounding boxes. Where are the cats? (YOLO is the canonical fast detector, arxiv 1506.02640.)
  • Semantic segmentation — a label for every pixel. Which pixels are "cat"? (U-Net, arxiv 1505.04597.)
  • Instance segmentation — like semantic, but distinguishes individual objects. Cat #1 vs cat #2.
  • Image generation — start from noise, end at an image. (Diffusion models, Ho et al. 2020, arxiv 2006.11239.)
  • Captioning & VQA — image in, language out.

The grid on the right shows six small previews — same input image, six different output shapes. Hover the buttons to see what each loss is measuring.

Pintea calls vision mathematically involved for a reason: it sits at the intersection of linear algebra (convolutions, transforms), probability (likelihoods, priors), optimization (gradients, regularization) and computer science (data structures, parallelism). All of those pieces show up in every paper.

Quick check

What output shape does semantic segmentation produce?

Semantic segmentation gives a per-pixel label. The network output is the same height and width as the input — just with channels for class probabilities instead of RGB.

Section 3 · the workhorse · ties to lesson 4

CNNs are still the baseline

Before we leave the safe ground of convolutional networks, remember why they work for images. Three properties: local connectivity (a pixel cares mostly about its neighbours), weight sharing (the same edge detector is useful in every patch of the image), and hierarchy (early layers find edges, middle layers find textures, late layers find object parts).

A convolution is just a small matrix — a kernel — sliding across the image. At each location it computes a weighted sum of the pixels under it. The output is a feature map: a new image where each value says how strongly that kernel responded at that location.

Drag the kernel slider on the right. Watch a 3×3 filter sweep across an 8×8 image and fill in the output map. Switch between an edge detector, a blur, and a sharpen kernel — each produces a wildly different feature map from identical pixels.

Why CNNs scale. One 3×3 kernel has nine weights. A fully-connected layer between two 224×224 images would have 2.5 billion. Weight sharing is what makes vision feasible — and the translation equivariance you get for free is a real inductive bias, not just a parameter saving.
Lesson 4 covered CNNs in depth. Treat this as a recap: it is the baseline that every model in the rest of this lesson is trying to beat — or at least match without using convolutions at all.

Section 4 · slide 18

Self-supervised learning

Labels are expensive. Pixels are free. Self-supervised learning exploits that asymmetry: you make up a pretext task whose answer you already know from the image itself, train on millions of unlabelled photos, then fine-tune on the small labelled set you actually care about.

Three families dominate:

  • Rotation prediction. Rotate the image by 0°, 90°, 180° or 270°. Ask the network which rotation was applied? (Gidaris et al. 2018). The only way to win this game is to learn what objects look like upright.
  • Masked patch reconstruction. Hide 75% of the image patches and ask the network to fill them in (MAE, He et al. 2021). Forces a global understanding of the scene.
  • Contrastive learning. Two augmented views of the same image should land close in feature space; views of different images should land far apart (SimCLR, MoCo, DINO). No labels — just a similarity loss.

The demo shows a single image rotated by a random angle. Pick the rotation you think was applied — that is exactly the supervision signal a self-supervised model trains on, billions of times.

Limitations Pintea highlights. Self-supervision is not magic. Pretext tasks can teach shortcut features — the model learns artefacts of the augmentation, not real semantics. It still needs massive compute and massive data to be competitive with supervised pre-training, and the gap to fully supervised models is real on fine-grained tasks.

Quick check

What does masked patch reconstruction force the network to learn?

Hiding 75% of patches means local features are not enough — to fill in the missing pieces the model has to reason about the whole scene. That global representation is what transfers to downstream tasks.

Section 5 · slides 19–20 · NeurIPS 2021

MLP-Mixer — back to linear layers

After ConvNets and Transformers, a 2021 paper from Google asked an awkward question: do we actually need convolutions or attention? MLP-Mixer (Tolstikhin et al., NeurIPS 2021, arxiv 2105.01601) answered "no" — at least if you have enough data.

The recipe is almost embarrassingly simple. Cut the image into a grid of patches. Flatten each patch into a token. Then alternate two kinds of MLP:

  • Token-mixing MLP. Mix information across patches but within each channel. This is the rough analogue of attention or convolution.
  • Channel-mixing MLP. Mix information across channels but within each patch. This is the rough analogue of a 1×1 convolution.

That's it. Two MLPs, alternated. No attention, no convolution. On ImageNet with enough pre-training, it matches ConvNets and Transformers.

The animation shows a 6×6 patch grid being processed in turn — first the rows light up (token mixing), then the columns (channel mixing). The two operations alternate.

Why this matters. Mixer is a clean existence proof: the inductive biases of convolution and attention are useful, but not strictly necessary. Given enough data, a stack of plain linear layers can catch up.
Limitations from the slide. Mixer "does not solve the computational costs of transformers" — token-mixing across all patches still scales poorly. And it "still requires very large datasets to train on". Without ConvNet-style inductive biases, you have to pay the data tax.

Section 6 · slides 21–23 · ICML 2024

Mamba — state-space models

Transformers are quadratic in sequence length. For long contexts — long videos, high-resolution images, multi-page documents — that quadratic cost hurts. Mamba borrows from control theory: a system has a hidden state that evolves over time as new inputs arrive.

# Discrete state-space model h_t = A_t · h_{t-1} + B_t · x_t y_t = C_t · h_t

Per the slide, the symbols are:

  • x_t — the input sequence
  • y_t — the output sequence
  • h_t — the hidden representation
  • A_t — the transition matrix for the hidden state
  • B_t — the contribution of the input to the hidden state
  • C_t — the transition matrix for the output
  • Δ_t — the discretization function (turns the continuous-time ODE into a discrete step)

The selective part of selective state-space models (Gu & Dao, 2023) lets A_t, B_t and Δ_t depend on the input x_t — so the model can choose what to remember and what to forget at each step. That selectivity is what makes Mamba competitive with Transformers, while keeping inference linear in sequence length.

Drag the input slider on the right. Watch h_t evolve over 30 time-steps. A controls how much of the past leaks forward; B controls how strongly each new input pushes the state.

Linear vs quadratic. A Transformer attending over N tokens does O(N²) work. A state-space model rolls a hidden state forward in O(N). For sequences of tens of thousands of tokens, that gap is everything.

Section 6b · slides 22–23

Vision Mamba — and its critics

Mamba was originally 1D — built for language and audio. Vision Mamba (L. Zhu et al., ICML 2024, arxiv 2401.09417) extends the scan to 2D images: split the image into patches, then sweep the state-space model across them. Patch order matters, so they scan in multiple directions and merge.

The pitch from the slide:

  • "Extended from 1D to 2D."
  • "Suitable for long distance dependencies, computationally more efficient than transformers."

That second point is the real claim. On a 1024×1024 image, a Vision Transformer's attention matrix has roughly a million squared entries. A state-space sweep is linear.

The push-back: MambaOut. W. Xu et al., "MambaOut: Do we really need mamba for vision?" (CVPR 2025), argue the answer is not really. Their experiments suggest that for image classification — where long-range dependencies are limited and the input is not autoregressive — a plain ConvNet or simple gated CNN matches Vision Mamba without the complexity. State-space models shine on long sequences, but most vision tasks aren't long sequences in the language-model sense.

The animation on the right shows the 2D scan order Vision Mamba uses — a snake-like sweep that turns a grid of patches into a 1D sequence the state-space model can roll across.

Quick check

The MambaOut paper's main argument is that Mamba is unnecessary for vision because:

Mamba's selectivity wins on long causal sequences. A 224×224 classification head looks at the whole image at once and doesn't need that. The MambaOut authors show plain ConvNets match Vision Mamba on standard benchmarks — so the linear-time advantage isn't being cashed in.

Section 7 · slide 24

LLMs meet vision

The dominant trend right now is multimodal: glue a vision encoder onto a language model and let them share a representation. Three architectures define the space:

  • CLIP (Radford et al. 2021). Train an image encoder and a text encoder side by side on 400M image-caption pairs. Loss: matching captions land close in a shared embedding space; mismatched ones land far. Result: a single embedding space where the word "dog" sits near pictures of dogs.
  • LLaVA / GPT-4V / Gemini. Take a frozen CLIP-style vision encoder, project its outputs into the token space of a language model, and let the LLM treat the image as a few hundred extra tokens. The LLM does the reasoning.
  • Diffusion + text encoders. Stable Diffusion, DALL·E, Imagen — text-conditioned image generation. The text encoder steers the noise-to-image process.

Drag the text label on the right. Images move toward or away from it depending on how close their embeddings are to the prompt. This is exactly what CLIP-style retrieval does, just in 2D for clarity.

Why this is a shift. For decades, vision and language were separate fields with separate datasets and separate evaluation. Multimodal models force a shared representation — and that shared representation, not any one architecture, is what enables zero-shot classification, open-vocabulary detection, and conversational image understanding.

Quick check

What is CLIP's training objective?

CLIP uses a contrastive loss. There is no decoder. The whole training signal is "matching pairs close, mismatched pairs far". That single objective produces a shared embedding space rich enough to do zero-shot classification by just comparing an image to the embedding of class-name prompts.

Section 8 · slides 25–26

Next trends

Two directions Pintea singles out as "what comes next" — both already happening at the major venues.

Physics-informed vision. Generative video models can produce striking clips, but they routinely violate physics: water flows uphill, objects pass through each other, lighting is inconsistent. The fix is to put a physics simulator in the loop: generate, simulate, check, regenerate. Foo et al., "Physical Simulator In-the-Loop Video Generation" (CVPR 2026) is one example. There is a CVPR'25 workshop dedicated to it: visionmeetphysics.github.io.

Embedded intelligence. A network trained on a dataset of static images is fundamentally passive. The next step is agents — robots, embodied avatars, simulated bodies — that learn by interacting with an environment. The agent moves, observes the consequences, and updates its model of the world. The supervision signal is the environment itself, not a human labeller.

The animation shows two agents — one in a static-image pipeline (passive), one in a closed loop with a simulator (active). Watch how the active agent corrects its own mistakes. The passive one doesn't know it made any.

The thread tying both together. Both trends move computer vision away from "predict labels for static images" and toward "reason about a world that responds". The world — physical or simulated — provides corrections that no labelled dataset can.

Check yourself

Q1. What was the famous mistake in Papert's 1966 "Summer Vision Project"?

The proposal assumed vision was a few weeks of student work. The semantic gap between brightness values and category labels turned out to be a sixty-year research program.

Q2. Which task outputs a class label for every pixel?

Detection produces bounding boxes; classification produces one label. Semantic segmentation labels every pixel — same height and width as the input.

Q3. Why are CNNs so much more parameter-efficient than fully-connected networks for images?

A 3×3 kernel has 9 weights regardless of image size. A fully-connected layer would have one weight per (input pixel × output pixel) pair — billions for high-res images.

Q4. The point of self-supervised pretext tasks like rotation prediction is to:

You already know which rotation you applied — the label is free. Train on millions of unlabelled images this way, then fine-tune on the small labelled set you actually have.

Q5. The two MLPs alternated inside an MLP-Mixer block are:

Token-mixing mixes across patches within each channel; channel-mixing mixes across channels within each patch. No attention, no convolution.

Q6. Mamba's state-space recurrence h_t = A·h_{t−1} + B·x_t has computational complexity:

A recurrence that rolls a fixed-size hidden state forward visits each token once, so the cost is linear. That is the main advantage over Transformer attention on long sequences.

Q7. In the state-space model, what role does the matrix A play?

Per the slide: A_t is the transition matrix for the hidden state. B_t mixes in the input and C_t projects to the output.

Q8. The MambaOut paper (CVPR 2025) argues that:

Their experiments show that on classification benchmarks — where there is no long autoregressive dependency to exploit — Vision Mamba's advantages don't materialise.

Q9. What is the training objective of CLIP?

CLIP pulls matching image-caption pairs together in embedding space and pushes mismatched ones apart. That single objective produces a shared image-text representation.

Q10. "Physics-informed vision" means:

A simulator provides corrections a labelled dataset can't — water flows downhill, objects don't overlap, light obeys conservation. The model generates, simulates, and regenerates.

Q11. "Embedded intelligence" in computer vision refers to:

"Embedded" here means embodied: a robot or simulated agent acts, observes consequences, and updates its world model. The supervision is the environment, not a labeller.

Q12. Which of these is the fastest object detector in standard practice?

YOLO ("You Only Look Once", arxiv 1506.02640) treats detection as a single regression problem and runs in real time. U-Net is for segmentation; Stable Diffusion is a generator.
