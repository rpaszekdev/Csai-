Lesson One

Why one neuron isn't enough.

A perceptron draws a single straight line to separate data. The XOR problem refuses to be solved that way. You'll see why — and how a hidden layer fixes it.

Slides by Dr. Juan Sebastián Olier
Department of Cognitive Sciences and AI · Tilburg University
Section 1 · slide 2

The Perceptron

A perceptron takes inputs, multiplies each one by a weight, adds a bias, then passes the total through an activation function. The classic perceptron uses a step function: output 1 if the total ≥ 0, otherwise 0.

The error (target − output) tells us how to adjust the weights. This is the perceptron learning rule:

Δw = η · error · input
For each weight, the update is the learning rate η times the error times the input that weight was connected to. The bias updates the same way — just treat it as an input that's always 1.

Press ▶. Each input pulses along its weight, they meet at Σ, then f squashes the total into an output.

A perceptron is a linear classifier. Its decision boundary is always a straight line in 2D (or a flat plane in higher dimensions). It's essentially the same idea as logistic regression.

Quick check

If we doubled all the weights but kept the inputs the same, what happens to the decision boundary?

Scaling the weights scales the total, but the zero crossing stays put. Bigger weights mean more confidence, not a different decision.

Section 3 · slides 7–9

Hard case: XOR

XOR outputs 1 when exactly one input is 1. The two 1-points sit on opposite corners of the square. So do the two 0-points.

Try to draw one line that separates them. You can't — no single straight line can split the 1's from the 0's in this layout.

Quick check

Why does XOR break the perceptron?

A perceptron's decision boundary is always a straight line. XOR needs a curved or broken boundary. One perceptron can't make one.

Section 4 · slides 10–13 · the key idea

The fix: a hidden layer

Use two perceptrons instead of one. Call them p₁ and p₂. Feed their outputs into a third perceptron.

The first two perceptrons reshape the data into a new space. In that new space, a single straight line is enough.

Press ▶. The four points rise out of the flat plane. The two 1-points go up; the two 0-points stay down. A flat plane now slides between them.

This is what a hidden layer does. It reshapes the input into a new space where a straight boundary finally works. Every deep network is this trick, repeated.

Section 5 · slide 16

A layer is stacked perceptrons

One perceptron gives one output. Put several of them side by side, sharing the same inputs. Now you have a layer with many outputs. Each neuron keeps its own weights and bias.

Press ▶ to add neurons to the layer.

A whole layer collapses into one formula: o = f(Wx + b). W is the weight matrix, b is the bias vector. This is why GPUs matter — matrix multiplication is the one thing they do very fast.
a layer of perceptrons
Section 6 · slides 19–23

Weights bend space

A weight matrix doesn't just multiply numbers. It bends the grid. Every point in the plane moves to a new place, following one consistent rule.

Drag the slider. At t = 0 you see the original grid. At t = 1 you see the grid after the transformation y = Wx. Watch the red and green arrows — those are the new directions of the axes.

But bending alone isn't enough. Try this matrix on the four XOR points:

(0,0) → (0,0)   (1,0) → (1,−1)   (0,1) → (−1,1)   (1,1) → (0,0)

Still four distinct points. Still not linearly separable. The matrix just rotated and mirrored them. The problem is unchanged.

The missing piece is the activation function. The step function maps any negative number to 0. So (1,−1) becomes (1,0) and (−1,1) becomes (0,1). And (1,1) stays at (0,0). Now two points sit on top of each other — and one straight line can separate them.

This is the real trick. A stack of matrix multiplications without activations collapses into one matrix multiplication — still linear, still just a single straight line. The non-linear activation between layers is what lets a network bend the problem into a new shape. No activation, no deep learning.

Section 7 · slides 25–26

The Multilayer Perceptron

Stack an input layer, one or more hidden layers, and an output layer. Connect every node to every node in the next layer. That's a Multilayer Perceptron, or MLP.

Each layer does the same thing: take the previous layer's output, multiply by weights, add bias, activate. Press ▶ to watch a signal travel through.

The same layer goes by three names: dense, fully connected, or linear. In PyTorch you'll see it as nn.Linear. All the same thing.
An MLP with one hidden layer and enough neurons can approximate any continuous function. In practice we use many thin layers — easier to train, better generalization.
But how do we train it? The perceptron update rule Δw = η · error · input only works when you know the target for that neuron. For a hidden neuron, you don't — you only know the target at the output. So the perceptron rule breaks for MLPs. The fix is backpropagation — coming in the next lesson.

Loss: how wrong are we?

The loss (also called error or cost) measures how wrong the network's output is. Training a neural network means finding weights that make the loss as small as possible.

Section 9 · slides 29–33

Same network, many jobs

Under the hood, every task is either classification (which category?) or regression (what number?). You just change what the outputs mean and which loss you measure.

  • Classification — one output per class. Target is one-hot: exactly one output should fire.
  • Compression (autoencoder) — squeeze the input through a narrow hidden layer, then rebuild it on the other side. What survives the squeeze is what actually mattered.
  • Time-series prediction — past values in, next value out. A regression task.
  • Multi-head — one network, several outputs at once: what (classification) and where (regression).

Check yourself

Q1. A single perceptron's decision boundary is always:

A perceptron computes w·x + b = 0, which is always a straight line in 2D or a flat plane in higher dimensions.

Q2. Why can't a single perceptron solve XOR?

No single straight line can separate the two 1-points from the two 0-points.

Q3. What does a hidden layer do?

The hidden layer transforms the input. In the new space, the final layer can finally find a straight boundary.

Q4. Why do we write a whole layer as Wx + b instead of looping over neurons?

Matrix multiplication runs massively in parallel on GPUs. That's what makes modern deep learning feasible.

Q5. An autoencoder's hidden layer is narrower than the input. This forces it to:

Information has to squeeze through the bottleneck. The network learns to keep what's important and throw away the rest.

Q6. The loss function exists to:

Loss is a single number saying how wrong you are. Its gradient tells the weights which way to move.

Q7. What happens if you stack several layers without a non-linear activation between them?

A chain of matrix multiplies is still one matrix multiply. Without non-linearity, depth adds nothing — you're back to drawing a single straight line.

Q8. Why can't the perceptron learning rule Δw = η · error · input train an MLP?

The rule needs an error for each neuron. In a hidden layer there's no target — only the final output has one. That's what backpropagation solves.

Q9. At the core, every neural network task is either:

Every output is either a category (classification) or a number (regression). Everything else is a combination of these two.
