Section 1 · slides 2–3

Where we left off

forward pass weights prediction

Last lesson we found that XOR needs two perceptrons in a hidden layer plus a third to combine them. The architecture works — but the old perceptron rule Δw = η · error · input needs a target for each neuron.

For the hidden neurons, we don't have a target. Only the output has one.

The core question: if we can't directly tell each hidden neuron "here's what you should have produced," how can we know which way to nudge its weights?

Answer: the network's output error gets propagated backward, neuron by neuron, using the chain rule.

Section 2 · slide 4

A network is a function

network = composition of functions

Here's the frame that makes everything click: a neural network is a function y = f(x; W) where W is all the weights.

Somewhere out there is a true function that maps inputs to the correct outputs. We don't know its form — we just have examples (training data).

Training means adjusting W until our network's function looks like the true one, at least on our examples.

A network with enough neurons is a universal function approximator — it can get arbitrarily close to any continuous function. Training is the search for the right weights.

Section 3 · slide 6 · the key idea

Gradient descent

steepest descent learning rate

Think of the loss L as a landscape. Every choice of weights is a point in this landscape. The height at that point is how bad the network is doing.

Training means walking downhill, toward the lowest point.

The gradient ∇L points uphill — the steepest direction of increase. So we move in the opposite direction:

W ← W − η · ∇L

η is the learning rate — how big a step to take. Too small = slow; too big = you overshoot the minimum.

Try it on the right. Click anywhere on the surface to drop a ball there. It follows the gradient downhill. Start from different places — you'll land in different local minima. Real networks face this problem in thousands of dimensions.

Section 4 · slides 7–8

What is optimization?

minimize loss over weights

Optimization is: picking the best parameters, by some criterion, out of all alternatives.

For a neural network, those three pieces are:

Parameters — all the weights and biases in the network.
Criterion — the loss function.
Alternatives — every possible value those weights could take.

That "every possible value" is a huge space. A modern network has millions or billions of parameters. We can't enumerate all choices — we need a smarter search. Gradient descent is that search.

Each step uses the gradient to pick a slightly better set of weights than the current one. Repeat many times, and you arrive somewhere decent.

Section 5 · slide 14

The chain rule

derivative of compositions

To get the gradient, we need derivatives. And because layers are nested functions, we need the chain rule.

If L = f(a) and a = g(z), then:

∂L/∂z = (∂L/∂a) · (∂a/∂z)

In words: how much does L change when z changes? It's the product of two smaller sensitivities — how L depends on a, times how a depends on z.

In a neural network the chain keeps going:
loss → output activation → weighted sum → weights.
Each arrow is a derivative. The chain rule multiplies them all.

Section 6 · slides 17–22

Gradient of the output layer

loss vs prediction activation derivative

For the output layer n, applying the chain rule to ∂L/∂W gives us three terms:

∂L/∂W = (∂L/∂a) · (∂a/∂z) · (∂z/∂W)
  • ∂L/∂a — how the loss changes when the output changes. The error component.
  • ∂a/∂z — the derivative of the activation function.
  • ∂z/∂W — simple: it's just the input to this layer.

Group the first two and give them a name: δ (delta).

δ = (∂L/∂a) · (∂a/∂z)   ⇒   ∂L/∂W = δ · input
δ is the "error signal" of the layer. It measures how much the layer's weighted sum z should shift to reduce the loss. Multiply δ by the layer's input and you get the weight gradient.

Section 7 · slides 28–32

Sending error backward

send error one layer back at a time

Good — we have δ for the output layer. What about the hidden layers?

Another round of the chain rule gives the rule for every previous layer:

δl−1 = (Wl)ᵀ · δl · f'(zl−1)

Each layer's error is the next layer's error, multiplied by the same weights that carried the signal forward (transposed), and then shaped by the derivative of this layer's activation.

Backprop is the forward pass in reverse.
Forward: input flows left → right through the weights, becoming an output.
Backward: the output's error flows right → left through the same weights, becoming a per-layer error signal.

Press ▶ on the right — first watch the signal go forward (blue pulses). Then the error comes back (red pulses), through the same connections, in reverse.

Section 8 · slide 34

Update the weights

each weight nudged by its gradient

Now we have a gradient for every layer. Apply the same rule as gradient descent, layer by layer:

Wl ← Wl − η · ΔWl
with   ΔWl = (al−1)ᵀ · δl

And the same for the bias:

bl ← bl − η · δl

That's it. That's the whole training step:

1. Forward pass — input through the network, get the output.
2. Loss — compare output to target.
3. Backward pass — propagate δ back through all layers.
4. Update — adjust every weight by its gradient × learning rate.
Repeat thousands of times.

In libraries like PyTorch this all happens automatically via autograd. But under the hood it is exactly the chain rule, layer by layer.

Section 9

Why we dropped the step function

step → sigmoid differentiable nonlinearity

The classic perceptron used a step function: output 1 if the sum ≥ 0, else 0. Simple. But it has a fatal problem for backprop.

The step function has no derivative at its jump — and zero derivative everywhere else. Every term in the chain rule gets multiplied by zero. No gradient, no learning.

So deep learning uses smooth activations:

  • Sigmoid — σ(z) = 1 / (1 + e⁻ᶻ). Squashes into (0, 1). Smooth everywhere.
  • Tanh — similar, but output in (−1, 1).
  • ReLU — max(0, z). Not smooth at 0, but its left-and-right derivatives are well-defined (0 and 1), and that works fine in practice.

Your choice of activation decides what f'(z) in the backprop equation actually is. Sigmoid's derivative is σ(z)·(1 − σ(z)), for example — elegant, easy to compute from the sigmoid itself.
