Section 1 · motivation

Why not just use a fully-connected net?

FC blows up spatial structure wasted

An MLP flattens an image into a long vector, then multiplies by a giant weight matrix. For a modest 200×200 RGB image, the first layer alone needs 120,000 inputs — and if the next layer has 1,000 neurons, that's 120 million weights. One layer.

Worse: the MLP does not know two pixels are neighbours. If the cat shifts ten pixels to the right, every weight must be relearned.

Convolutions assume two things about images:
Locality — the useful patterns are small and local (an edge, a corner, a texture).
Translation invariance — a cat is a cat whether it's in the top-left or the bottom-right.

From those two assumptions: share one small filter across the whole image. Suddenly you have a few dozen weights instead of millions — and the network sees shifted cats for free.

Section 2 · the core operation

What a convolution actually does

kernel slide sum

A filter (also called a kernel) is a tiny grid of numbers — often 3×3. You slide it across the image. At each position: multiply element-wise, then sum. That single number becomes one pixel in the output.

out[i, j] = Σ kernel[k, l] · image[i+k, j+l]

The filter looks for one pattern. An edge-detector filter produces big numbers where the image has an edge and small numbers in flat regions. A blur filter averages neighbours.

Picking filter weights by hand is painful — so a CNN learns the filters from data, the same way an MLP learns its weights: through backpropagation.

Watch the 3×3 window on the right step across the input. The cell it writes to in the output is the sum of the nine products. Try different presets.

Section 3 · knobs on the operation

Kernel size, stride, padding

step size output shape padding

Three choices shape the output:

  • Kernel size — usually odd numbers (3, 5, 7) so the filter has a natural centre. Bigger kernels capture bigger patterns but cost more.
  • Stride — how many pixels you jump between applications. Stride 1 = check every position. Stride 2 = skip every other one, output shrinks by half.
  • Padding — zeros added around the border so the filter can reach the edges. Without it, the output shrinks a little every layer.
Wout = (Win − F + 2P) / S + 1

F = filter width, P = padding, S = stride. The same formula works for height.

Example from the lecture. Input 5×5, filter 3×3, padding 1, stride 2. Then (5 − 3 + 2)/2 + 1 = 3. The output is 3×3 — matching what you see on the right.

Section 4 · downsampling

Pooling — summarise and shrink

max / avg over a window

After a convolution, we often downsample. The idea: if a pattern appears somewhere in a small region, we don't need to know the exact pixel. Keep the summary, throw away the precise position.

The two common flavours:

  • Max pooling — take the largest value in each window. "Did the pattern fire anywhere in here?"
  • Average pooling — take the mean. Smoother, softer.
Why it helps. Pooling gives the network a little translation tolerance — shift the input by one pixel and the pooled output barely changes. It also cuts the number of values the next layer has to process.

You can also downsample within the convolution, by using stride > 1. Modern networks often mix both tricks.

Section 5 · the whole pipeline

Stacking it all: a CNN

conv → ReLU → pool repeat

A typical classification CNN is a repeating pattern:

conv → ReLU → pool   ⟶   conv → ReLU → pool   ⟶   …   ⟶   flatten → dense → softmax

Two things happen as you go deeper:

  • The spatial size shrinks (pooling + strided convs).
  • The number of channels grows — early layers fire on simple edges, middle layers on textures, later layers on object parts, and the deepest on whole concepts like "cat".

At the end, the feature maps are tiny but deep. You flatten them into a vector and hand it to a small MLP that outputs one number per class.

A filter is never just 2D. If the input has C channels, each filter has C channels too. The filter looks through all input channels at once and produces one output channel. Use K filters, get K output channels.
Counting parameters. For a 4×4 filter on an RGB input (3 channels), with 32 output filters: 4 · 4 · 3 · 32 + 32 = 1568 parameters (the last +32 is one bias per filter). Independent of image size.

Section 6 · slides 17–31 · variations

Dilated · 3D · Transposed

dilated 3D transposed

The basic recipe has a few useful variants:

Dilated (atrous) convolution — keep the 3×3 filter, but spread its samples out with gaps. A 2-dilated 3×3 filter still has 9 weights, but now covers a 5×5 area. Stack a few of them and the receptive field grows exponentially without losing resolution.

3D convolution — slide the kernel in a third dimension too. Used for video (time is the third axis), medical volumes (CT, MRI), and point clouds.

Transposed convolution — the reverse move. Instead of shrinking, it upsamples: each input pixel is multiplied by the whole kernel and placed into a larger output. The network learns how to enlarge.

Where transposed convs show up. Encoder-decoders (compress then reconstruct), U-Net for segmentation, GANs that generate images from noise, and colorising greyscale photos.

Section 7 · depth and identity

1×1 convolutions & residual connections

1×1 mixes channels ResNet skips

A 1×1 convolution only looks at one position — but it sees every channel at once. It mixes channels together without touching space. Two common uses:

  • Channel compression — go from 256 channels down to 64 with a few 1×1 filters. Cheap way to shrink.
  • Channel mixing — blend different feature maps into new combinations.

Now the ResNet trick. He et al. (2016) noticed that making a plain CNN deeper (56 vs 20 layers) made it worse. Gradients vanished before reaching early layers.

Their fix: the residual block. Alongside the usual conv-ReLU-conv, add a shortcut that carries the input unchanged, and add it to the block's output:

y = F(x) + x
Why it works. If the block can't learn anything useful, F(x) can just go to zero and the network passes x through untouched. Gradients also flow backward through the shortcut — no vanishing. That option to do nothing is what lets ResNets go 100+ layers deep.

When the block changes the number of channels, the shortcut needs a 1×1 convolution so its shape matches the output — and the two can be added.

Section 8 · where CNNs are used

Applications

classification detection segmentation

The same backbone — stacked convs producing feature maps — powers a surprising range of tasks:

  • Image classification — one label per image. The classic CNN job.
  • Object detection — what is in the image and where. Produces bounding boxes + class labels.
  • Segmentation — assign a class to every pixel. Uses encoder-decoder shapes like U-Net.
  • Generation — upsample from a compressed code back to an image (autoencoders, GANs).

For detection, two families dominate:

Region-based (R-CNN, Faster R-CNN) — first propose candidate regions, then classify each one. Two-stage, accurate, slower.

YOLO — You Only Look Once — split the image into an S×S grid. Each cell predicts, in one pass, both what's there and a few candidate bounding boxes. Overlapping boxes are cleaned up by non-max suppression. Single-shot, fast enough for real time.

Others worth knowing: SSD, RetinaNet, and increasingly transformer-based detectors like DETR.
